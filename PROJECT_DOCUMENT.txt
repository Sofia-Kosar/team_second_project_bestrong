━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PoC (Proof of Concept) застосунок для:
  - Автоматичного зчитування PDF-документів (інвойси, чеки, прайс-листи)
  - OCR через Azure AI Document Intelligence
  - Аналізу цін за допомогою LLM моделі GPT-4o
  - Збереження результатів у Azure Blob Storage у форматі JSON
  - Push-нотифікацій у Discord та Slack при появі нового результату

Команда: DevOps Engineering Team 2
Завдання: IaC + CI/CD + Azure Function + Notification System

5.1 Функція 1: process_pdfs
────────────────────────────
  Тригер:    Timer (розклад: "0 */5 * * * *" = кожні 5 хвилин)
  Параметр:  run_on_startup=False (не запускається при деплої)

  Алгоритм:
  1. Підключається до File Share через ShareServiceClient
  2. Переглядає всі файли в кореневій директорії
  3. Пропускає файли що не є .pdf
  4. Пропускає PDF якщо вже є відповідний .json в Blob (уникає дублів)
  5. Завантажує PDF bytes
  6. Викликає _build_result() → повертає фінальний dict
  7. Серіалізує в JSON та зберігає в Blob

  Функція _build_result():
  ┌── Document Intelligence ──────────────────────────────────────────┐
  │ Модель: prebuilt-layout                                            │
  │ Вхід:   PDF bytes (bytes_source)                                   │
  │ Вихід:  page_count, content (текст), tables, key_value_pairs       │
  └───────────────────────────────────────────────────────────────────┘
  ┌── OpenAI GPT-4o ───────────────────────────────────────────────────┐
  │ Модель: gpt-4o (через platform.openai.com)                         │
  │ Вхід:   Перші 4000 символів тексту + системний промпт              │
  │ Формат: response_format={"type": "json_object"}                    │
  │ Вихід:  prices[], highest_price, lowest_price, summary             │
  └───────────────────────────────────────────────────────────────────┘

  Структура фінального JSON:
  {
    "file_name": "invoice.pdf",
    "processed_at": "2026-03-15T12:00:00+00:00",
    "ocr_result": {
      "page_count": 2,
      "content": "текст документу...",
      "tables": [{ "rows": 6, "cols": 4, "cells": [...] }],
      "key_value_pairs": [{ "key": "Total", "value": "$471.18" }]
    },
    "ai_analysis": {
      "model": "gpt-4o",
      "price_analysis": {
        "prices": [
          { "value": 471.18, "currency": "USD",
            "context": "Invoice Total", "classification": "HIGH" },
          { "value": 12.99,  "currency": "USD",
            "context": "Unit price", "classification": "LOW" }
        ],
        "highest_price": { "value": 471.18, "currency": "USD", "context": "..." },
        "lowest_price":  { "value": 12.99,  "currency": "USD", "context": "..." },
        "summary": "Invoice for sports equipment..."
      }
    }
  }

5.2 Функція 2: notify_result
──────────────────────────────
  Тригер:    Blob (path="results/{name}", connection="AzureWebJobsStorage")
  Запуск:    Автоматично при появі БУДЬ-ЯКОГО файлу в container "results"

  Алгоритм:
  1. Читає blob як JSON
  2. Витягує file_name, processed_at, ai_analysis.price_analysis
  3. Формує URL: https://stbesstrongocrdev.blob.core.windows.net/results/файл.json
  4. Викликає _format_message() → стандартизований dict
  5. Якщо DISCORD_WEBHOOK_URL не порожній → _send_discord()
  6. Якщо SLACK_WEBHOOK_URL не порожній → _send_slack()

  Discord повідомлення (embed):

  │ 📄 PDF Processed: invoice.pdf              │

  │                                            │
  │ Highest Price  │ Lowest Price              │
  │ $471.18 USD    │ $12.99 USD                │
  │ Invoice Total  │ Unit price                │
  │                                            │
  │ Processed At: 2026-03-15T12:00:00+00:00    │
  │ Result JSON: https://...results/invoice... │
